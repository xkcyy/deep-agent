# 实施 Backlog（按优先级）- 需求说明

## 目标与范围
- 目标：对全量功能进行分级拆解，便于排期与交付。
- 范围：列出 P0/P1/P2 的任务项及完成标准。

## Backlog 列表
- P0 登录与初始加载：登录成功进入主界面；配置/会话/资源根成功拉取或可“跳过”；失败明确提示并可重试。
- P0 对话输入与发送：文本/附件发送，loading/停止生成；错误可重试/复制请求；快捷键生效；发送后消息正确渲染。
- P0 模型配置与切换：设置可增改删模型；输入框可选模型并在消息标注；调用超时/401 提示并可跳转设置。
- P0 资源管理器与上下文绑定：文件树懒加载；节点可多选并绑定到当前会话；清空/恢复选择可用；预览正常；刷新可重新扫描固定目录。
- P0 工具注册与调用路由：工具清单可启用/禁用；发送后按规则/模型路由工具链；步骤/进度可见；失败自动重试一次，仍失败退化为模型答复并提示缺失数据。
- P0 主窗口与结果展示：结果默认开新卡片；多 Tab 可关闭/固定/重排；工具进度与错误展示；并行任务状态可见。
- P1 会话列表与历史折叠：按时间分组，近期展开，远期折叠；“加载更多”分页；搜索/筛选；删除/重命名需确认；切换同步对话区与主窗口。
- P1 提示词模板管理：模板增改删与预览；变量占位示例；对话前可选择模板，发送前可查看合成结果；删除提示引用影响，热更新生效。
- P1 会话与资源上下文可视化：每条消息可查看“上下文来源”；“清空上下文”“恢复上次选择”可用；长会话折叠历史，滚动加载。
- P1 导出与复制：消息/卡片一键复制；表格/图表导出 CSV/PNG；对话导出 Markdown 含模型/工具摘要与资源来源；操作后提示成功。
- P2 观测与故障提示（轻量）：基础日志/指标开关；显示请求耗时与工具步骤耗时摘要；统一错误提示（含追踪 ID）；用户反馈入口收集描述与截图。

## 验收标准
- 每个 Backlog 项均有可验证的完成标准，并与对应功能需求文档一致。
- P0 项完成后，可实现基本对话、模型切换、资源上下文、工具调用与结果展示的闭环。
