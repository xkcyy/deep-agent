# 观测与故障提示（轻量）- 需求说明

## 目标与范围
- 目标：提供基础可观测与统一故障提示，便于用户感知性能与问题。
- 范围：请求耗时与步骤耗时摘要、错误提示样式、追踪 ID 展示、用户反馈入口。

## 用户故事
- 作为用户，我想了解请求大致耗时和工具步骤耗时。
- 作为用户，我遇到错误时能看到统一的提示和追踪 ID，并能提交反馈。

## 主流程
1. 对话/工具请求完成后，在消息或主窗口显示耗时摘要（总耗时、步骤耗时）。
2. 出错时，用统一样式展示错误，包含追踪 ID、重试入口。
3. 页面提供“反馈”按钮，弹窗收集问题描述与截图（或自动带上追踪 ID）。

## 关键交互
- 耗时摘要显示在结果卡片尾部或消息角标；步骤耗时可展开。
- 错误提示组件：图标+短文案+追踪 ID，提供“重试/复制追踪 ID”。
- 反馈弹窗：文本输入、上传截图（可选）、附带最近一次追踪 ID。

## 边界与异常
- 若后端未返回追踪 ID，标记“无追踪 ID”，提示联系管理员。
- 反馈提交失败提示可重试，失败记录在本地缓存（可选）。

## 非功能
- 耗时渲染 < 100ms；错误提示渲染 < 200ms。
- 反馈提交接口超时 < 5s。

## 依赖
- 后端返回耗时与追踪 ID；反馈收集服务。

## 验收标准
- 正常请求显示耗时摘要，工具步骤耗时可展开。
- 错误提示统一、含追踪 ID，重试入口可用。
- 反馈入口可提交内容与截图，失败时有提示。
