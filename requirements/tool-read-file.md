# 需求：文件读取（read_file）UI

## 目标风格
- 暗色玻璃态卡片，轻渐变 + 霓虹描边，顶部状态色条；无位移抖动。
- 信息优先：路径/元信息胶囊，其次内容预览。

## 布局
- 头部：图标 + 路径（截断，14px 粗），状态徽标；顶部状态条。
- 元信息行：大小/mtime/MIME/二进制标识，使用浅底胶囊。
- 内容区：霓虹描边预览框，默认截断 6–8 行，提供“展开更多/收起”；二进制仅提示下载/打开。
- 空/等待：提示“暂无内容/等待返回…”，可附原始截断。
- 错误：红色轻提示，可折叠。

## 状态与交互
- pending/streaming：骨架 + 部分已返内容混排；保持可折叠。
- completed：显示完整/截断内容；二进制提示下载/打开（可选回调）。
- error/interrupted：保持内容可见，不抖动。
- 折叠：面板右上角 chevron，记忆展开；预览区独立展开/收起。

## 数据映射
- 路径优先级：`result.path` > `args.path` > `args.filepath` > `args.file_path` > `args.target` > `args.dst`，缺失用“(未返回路径)”.
- 内容：`result.content` 或 `result.chunks[]` 拼接；若 is_binary 为真，不展示内容。
- 元信息：`size`, `mtime`, `mime`, `is_binary`, `error`。
- 解析失败：将 result 作为文本显示并截断。

## 视觉细节
- 紧凑 padding（p-2~p-3），圆角 md，轻阴影；hover 仅变色不位移。
- 文字层级：标题 14px 粗；正文 12px；说明 11px。
- 预览框：边框 + 背景弱色，最大高约 240–260px，超出滚动。
