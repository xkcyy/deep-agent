# 需求：目录列举（ls）UI

## 目标风格
- 暗色玻璃态、轻渐变 + 霓虹描边，紧凑行；无抖动。
- 信息突出路径与类型，便于快速扫视。

## 布局
- 头部：工具名 + 目标路径，状态徽标 + 顶部状态条。
- 列表：紧凑行（12px），文件夹/文件图标带主色辉光，右侧 size/mtime，行 hover 描边高亮；列表可滚动。
- 空/等待：提示“无返回/等待返回…”，附原始截断（可选）。
- 错误：红色轻提示，可折叠。

## 状态与交互
- pending：骨架行若干，状态条主色。
- completed：最多展示 50 条；可后续扩展筛选/排序。
- error/interrupted：保持可视化，不抖动。
- 折叠：面板右上角 chevron；列表保持滚动不撑高。

## 数据映射
- 目标路径：`args.path` 或 `args.dir`，默认 "."。
- 列表：`result.items` 或 `result.entries`；字段 `name|path|is_dir|size|mtime`。
- 解析失败：原始文本截断展示。

## 视觉细节
- 行高紧凑；图标 14px；圆角 md；边框 + 轻阴影；hover 仅边框高亮，不位移。
