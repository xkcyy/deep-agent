# 需求：TODO 计划/任务（write_todos / todo 工具）UI

## 目标风格
- 暗色渐变 + 霓虹描边卡片；状态色条突出分组；高信息密度但清晰。
- 与聊天/ToolCallBox 统一风格，任务进度与优先级一目了然。

## 布局
- 头部：工具名 + 状态徽标 + 顶部状态条；可显示任务总数/完成数。
- 列表：紧凑行（12px），任务内容 + 彩色状态胶囊（pending/in_progress/completed），可选更新时间。
- 分组：按状态分组（进行中/待办/已完成），组标题 11px，分组卡带色条/辉光。
- 空/等待：提示“暂无任务/等待返回…”，附原始截断。
- 错误：轻量红色提示，可折叠。

## 状态与交互
- pending/streaming：骨架列表 + 已返回任务混排；顶部状态条闪动。
- completed：展示全部任务与状态；可折叠/展开组；可选“复制任务列表”。
- error/interrupted：保持列表可见，不抖动；显示错误文案。
- 折叠：面板右上角 chevron；列表区域可滚动，避免撑高。

## 数据映射
- 输入 args：可包含新建/更新的任务项（content/status/id），或操作意图。
- 输出 result：期望结构 `{ items: [{ id, content, status, updatedAt? }] }`；若返回平铺数组也支持。
- 状态字段：`pending | in_progress | completed`；缺失时默认 pending。
- 解析失败：将 result 作为文本截断显示并标注未结构化。

## 视觉细节
- 行内标签：状态标签用浅底深字（绿色完成、黄色进行、灰色待办）。
- 字号层级：标题 14px 粗；列表 12px；分组标题 11px；说明 11px。
- 间距：紧凑 padding（p-2~p-3），行距适中；圆角 md；阴影轻；hover 仅边框高亮，无位移。
